# Краткое описание

Когда TypeScript (npm) библиотека вырастает до огромных размеров, контролировать код, который используется, становится очень сложно.
Так как при экспортировании объектов из файла непонятно будут они использоваться в публичном апи или нет.
И если нет прямых использований внутри библиотеки или в тестах, или в демке библиотеки, такие экспорты могут висеть очень долго мертвым кодом

# Цель
Спроектировать утилиту, которая будет искать мертвый код в экспортируемых объектах и функциях TypeScript библиотеки.
Утилита должна выдать список файлов или объектов экспортиремых их файлов, которые не используются в публичном API библиотеки.
Если функция используется в тестах или в демке, но не используется больше нигде, включая публичный API библиотеки, то она считается мертвым кодом


# Входные данные
1. Библиотека TypeScript;
2. Публичный API библиотеки можно получить по файлам с экспортами в корне директории `src`, не только `index.ts`, так как могут быть именованные экпорты `src/utils.ts`;
3. Внешние библиотеки могут использовать только экспорты из публичного API библиотеки (запрет на использование внутреннего кода библиотеки в обход публичного API отслеживается другой утилитой);
4. Юнит тесты лежат в корне репозитория `./tests`;
5. Демка библиотеки лежит в каталоге `./demo`;
